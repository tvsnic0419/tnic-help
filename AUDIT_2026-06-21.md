# TNiC.help — Site Audit & High-ROI Upgrades

**Date:** 2026-06-21 · **Scope:** Full audit + implement top wins (trust/a11y, performance, SEO, conversion)

## Summary

The site is a mature, well-engineered Next.js 16 / React 19 platform. Type checking is clean, security headers and canonical-redirect middleware are in place, SEO infrastructure (dynamic sitemap, JSON-LD for WebSite/Organization/FAQ/SoftwareApplication/MedicalWebPage, per-route OG images, RSS discovery) is strong, and reduced-motion CSS exists. So the highest-ROI work is targeted fixes, not a rewrite. I implemented three safe, high-impact wins and verified them; the rest are prioritized recommendations below.

## Implemented (this session)

### 1. Fixed the hero LCP penalty — performance + SEO + conversion (highest ROI)
`components/sections/HeroSection.tsx` — the `<h1>` and lead paragraph (the page's largest contentful element) were rendered at `opacity: 0` via Framer Motion and only faded in after JavaScript hydrated. In server-rendered HTML the main headline was invisible, so Largest Contentful Paint couldn't register until hydration + animation completed. I set those two elements to `initial={false}` so they paint immediately at first byte. Net effect: faster measured LCP (a Core Web Vitals ranking + UX signal) and the value proposition is visible instantly, including to crawlers and slow connections. Below-the-fold entrance animations were left intact.

### 2. Added a real app-icon set — trust + installability + Lighthouse
Previously only `favicon.ico` existed and `manifest.ts` had **no** `icons`. Added a brand mark (emerald/cyan "orbit" motif matching the hero) as:
- `app/icon.svg` — crisp vector favicon (auto-wired by Next)
- `app/apple-icon.png` — 180×180 opaque Apple touch icon (auto-wired)
- `public/icon-192.png`, `public/icon-512.png`, `public/icon-maskable-512.png` — referenced from `manifest.ts` (`any` + `maskable` purposes)

Improves PWA installability, the Lighthouse PWA/best-practices score, and how the site looks when saved to a home screen or shared.

### 3. Tightened robots — SEO crawl hygiene
`app/robots.ts` — added `disallow: '/api/'` (cron/subscribe endpoints shouldn't be crawled or indexed) and an explicit `host`. Sitemap reference unchanged.

## Recommendations, prioritized by ROI

**P1 — React Hooks correctness debt (24 ESLint errors).** Pre-existing, not introduced here, and intentionally excluded from CI (the workflow notes "Lint has pre-existing debt"). They are real React 19 issues: `setState` called synchronously inside effects (cascading re-renders) across ~18 components, and "cannot access refs during render" in `HeroNetworkCanvas.tsx` and `useLabOrderWatcher.ts`. These cause wasted renders and potential hydration/perf bugs. Worth a dedicated cleanup pass.

**P1 — Reduced-motion coverage for Framer Motion.** The CSS `prefers-reduced-motion` block does not stop Framer's JS-driven animations; only `HallmarkCoverageGrid` currently checks `useReducedMotion()`. Hero and section entrances still animate for users who requested reduced motion (WCAG 2.3.3 / vestibular safety). Gate entrance animations behind `useReducedMotion()`.

**P2 — Measurement.** There is no analytics anywhere, which is consistent with the privacy-first brand ("No Third-Party Analytics") — but you cannot optimize conversion ROI you cannot measure. Recommend a privacy-respecting, cookieless, first-party option (e.g. Vercel Web Analytics or self-hosted Plausible) gated behind the existing consent banner. This is a product decision because of the brand tension.

**P2 — Content-Security-Policy.** Security headers are strong but there is no CSP. Adding one is a real trust/security win; it needs careful testing against Framer inline styles and any third-party image hosts.

**P3 — Image CLS check.** Product images already use `next/image` with remote patterns configured — confirm explicit `sizes`/dimensions on shop cards to avoid layout shift.

**P3 — Lint in CI as non-blocking.** Run `npm run lint` as an annotations-only CI job so new hook debt is surfaced on PRs without blocking deploys.
